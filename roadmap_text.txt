2.  Models — Introduction, Pretrained Usage & Code 
This section covers every model used in the project. For each model you will find: what it is, why we chose it,

  STEP 01  Data Acquisition   [ Weeks 1–2 ] 
  Before running any model, we need satellite imagery. We use two free data sources: Sentinel-2 (10m resolution, 13 
bands, from ESA) and Landsat-8/9 (30m resolution, from USGS). Google Earth Engine (GEE) is a cloud platform that 
gives free access to both, so we can download images directly without manual searching. 
 
  Google Earth Engine (GEE) — Data Platform 
What is it? 
GEE is not a model — it is a cloud-based geospatial platform that holds petabytes of satellite data. 
You write a short JavaScript or Python script to define your area, date range, and cloud filter, and it 
exports the image directly to your Google Drive or local machine. 
Why use it? 
It gives free access to Sentinel-2, Landsat-8/9, and hundreds of other datasets without 
downloading anything manually. It handles tiling, reprojection, and band selection automatically. 
Pretrained 
Usage 
No model to load. Just authenticate and run the export script. 
Documentation

  STEP 02  Cloud Detection   [ Weeks 2–3 ] 
  Cloud detection is a semantic segmentation task: for every pixel in the image, the model decides whether it is a cloud or 
clear sky. We use a U-Net architecture with a pretrained ResNet34 encoder. The encoder was trained on ImageNet and

already understands edges, textures, and shapes — making it excellent at detecting cloud boundaries even without 
satellite-specific training. 
 
  U-Net + ResNet34 (segmentation-models-pytorch) 
What is it? 
U-Net is an encoder-decoder segmentation network. The encoder (ResNet34) compresses the 
image into feature maps, and the decoder expands them back to a full-size mask. The encoder 
weights come pretrained from ImageNet — we use the decoder as-is for general cloud detection 
without any fine-tuning. 
Why use it? 
The pretrained ResNet34 encoder detects low-level features (edges, brightness gradients) that are 
exactly what separates clouds from land. U-Net's skip connections preserve spatial detail for sharp 
mask edges. 
Pretrained 
Usage 
The encoder loads ImageNet weights automatically via the encoder_weights='imagenet' argument. 
No checkpoint download needed.

  STEP 03  Cloud Removal (Inpainting)   [ Weeks 3–4 ] 
  After detecting which pixels are cloud, we need to reconstruct the missing surface information underneath. This is called 
inpainting. We use OpenCV's Telea algorithm — a classical method that fills masked regions by propagating information 
from surrounding clear pixels. For larger cloud regions, DSen2-CR (a deep learning model) gives smoother results. 
 
  OpenCV Inpainting — Telea Algorithm 
What is it? 
The Telea algorithm fills masked (cloud) pixels by computing a weighted average of known pixels 
in the surrounding neighborhood, working from the mask boundary inward. It runs per-band 
independently, so it works on any number of spectral bands. 
Why use it? 
Fast, no GPU needed, no model weights to load, and works well for small-to-medium cloud 
patches. Best choice when cloud coverage is under 40%. 
Pretrained 
Usage 
Classical algorithm — no pretrained weights. Just call cv2.inpaint(). 
Documentation 
OpenCV Docs: docs.opencv.org/4.x/df/d3d/tutorial_py_inpainting.html 
YouTube 
Tutorial 
YouTube: 'Image Inpainting with OpenCV in Python' — Nicolai Nielsen 
MODEL INPUT 
Single band image: (H, W) uint8, cloud mask: (H, W) uint8 (255=cloud, 0=clear), inpaint radius (int) 
MODEL 
OUTPUT 
Inpainted band: (H, W) uint8, cloud pixels replaced with estimated surface reflectance values 
 
Code Example (PyTorch): 
# pip install opencv-python numpy rasterio 
import cv2 
import numpy as np 
import rasterio 
 
def remove_clouds(image_path, mask, inpaint_radius=7): 
    with rasterio.open(image_path) as src: 
        meta = src.meta.copy() 
        cleaned = np.zeros_like(src.read(), dtype=np.uint8) 
        for b in range(src.count): 
            band = src.read(b+1) 
            # normalize to uint8 
            band_u8 = cv2.normalize(band, None, 0, 255, 
                                    cv2.NORM_MINMAX).astype(np.uint8) 
            # inpaint cloud pixels 
            cleaned[b] = cv2.inpaint(band_u8, mask, 
                           inpaint_radius, cv2.INPAINT_TELEA) 
    return cleaned  # shape: (bands, H, W) 
 
  STEP 04  Preprocessing & Spectral Indices   [ Week 5 ] 
  Before running change detection, we compute spectral indices that highlight specific land types. NDVI (Normalized 
Difference Vegetation Index) is high for healthy crops and low for bare soil or buildings. NDBI (Normalized Difference Built-
up Index) is high for concrete/rooftops. These indices give the change detection model much richer signals than raw pixel 
values alone. 
 
  Spectral Index Computation — NumPy / rasterio

What is it? 
Spectral indices are mathematical formulas applied band-by-band to the image array. No neural 
network is involved — just arithmetic on the pixel values using NumPy. NDVI uses the Near-
Infrared and Red bands. NDBI uses the Short-Wave Infrared and NIR bands. 
Why use it? 
Index maps compress complex multispectral data into single-band images that directly encode 
vegetation health or built-up density. This makes the job of downstream models much easier. 
Pretrained 
Usage 
No model. Pure NumPy computation. 
Documentation 
rasterio Docs: rasterio.readthedocs.io | NDVI on Sentinel-2: sentinel-
hub.com/develop/documentation/eo_products/Sentinel2EOproducts 
YouTube 
Tutorial

MODEL INPUT 
Cloud-free GeoTIFF T1 and T2 with NIR (B8), Red (B4), SWIR (B11), Green (B3) bands — shape 
(bands, H, W) 
MODEL 
OUTPUT 
NDVI array (H, W) float32 in [-1, 1] | NDBI array (H, W) float32 in [-1, 1] | MNDWI array (H, W)

Code Example (PyTorch): 
import numpy as np 
import rasterio 
 
def compute_indices(image_path): 
    with rasterio.open(image_path) as src: 
        # Sentinel-2 band order (1-indexed): B2=1,B3=2,B4=3,B8=4,B11=5 
        red  = src.read(3).astype(np.float32)  # Band 4 
        nir  = src.read(4).astype(np.float32)  # Band 8 
        swir = src.read(5).astype(np.float32)  # Band 11 
        grn  = src.read(2).astype(np.float32)  # Band 3 
 
    eps = 1e-10  # avoid division by zero 
    ndvi  = (nir - red)  / (nir + red  + eps)  # vegetation 
    ndbi  = (swir - nir) / (swir + nir + eps)  # built-up 
    mndwi = (grn - swir) / (grn + swir + eps)  # water 
    return ndvi, ndbi, mndwi

  STEP 05  Land-Use Change Detection   [ Weeks 6–8 ] 
  Change detection compares two co-registered images (T1 = before, T2 = after) at the pixel level and predicts which 
pixels have changed and which have not. We use ChangeFormer — a Siamese Transformer model pretrained on the 
LEVIR-CD and WHU-CD building change datasets. It takes both images simultaneously and outputs a binary change 
map. 
 
  ChangeFormer — Siamese Transformer for Change Detection 
What is it? 
ChangeFormer is a dual-branch (Siamese) network. Both branches share the same Transformer 
encoder (MixTransformer from SegFormer). Each branch processes one image (T1 or T2), and a 
difference module at the decoder detects changed regions. Pretrained weights are available on the 
LEVIR-CD dataset (building changes in urban areas). 
Why use it? 
Transformer encoders capture long-range spatial relationships — critical for detecting gradual 
land-use changes. ChangeFormer consistently outperforms CNN-based methods on public 
benchmarks. Pretrained on building-change data, it is highly relevant for detecting new 
construction on farmland. 
Pretrained 
Usage 
Download pretrained weights from the official GitHub repo. Load with torch.load(). 
Documentation

  STEP 06  Agricultural Land Segmentation   [ Weeks 8–9 ] 
  We need to know exactly which pixels were agricultural land in T1 (before). We use SegFormer, a transformer-based 
segmentation model from NVIDIA, pretrained on the ADE20K dataset (150 semantic classes including farmland, grass, 
and fields). We load the pretrained model from HuggingFace and run inference directly — no fine-tuning needed. 
 
  SegFormer-B4 (HuggingFace Transformers — PyTorch)

What is it? 
SegFormer is a hierarchical Transformer encoder (Mix Transformer, MiT) combined with a 
lightweight MLP decoder. The B4 variant provides a strong balance between accuracy and speed. 
Pretrained on ADE20K, it recognizes 150 land-cover classes including field, earth, dirt, grass, and 
farm categories that map directly to agricultural land. 
Why use it? 
HuggingFace makes it trivially easy to load SegFormer with one line. ADE20K includes agriculture-
relevant classes. The model generalizes well to satellite imagery because it was trained on diverse 
outdoor scenes. 
Pretrained 
Usage 
Weights download automatically from HuggingFace when you call from_pretrained(). No manual 
download needed.

  STEP 07  Building Detection on Farmland   [ Weeks 9–11 ] 
  We now detect which changed pixels (from Step 5) that overlap with agricultural land (from Step 6) contain buildings. We 
use two complementary approaches: SAM (Segment Anything Model from Meta) for zero-shot segmentation of any object, 
and YOLOv8-seg for fast instance-level building detection using SpaceNet pretrained weights. 
 
  SAM — Segment Anything Model (Meta AI) 
What is it? 
SAM is a foundation model for image segmentation trained on 1.1 billion masks. Given any image, 
SAM can segment every object automatically without any labels or fine-tuning. It operates in three 
modes: automatic (segments everything), point-prompted (click on a building), and box-prompted 
(draw a box). We use the automatic mode on changed areas to isolate building footprints. 
Why use it? 
SAM works zero-shot on satellite imagery without any training. It handles irregular building shapes, 
partial rooftops, and high-density construction that rule-based methods miss. 
Pretrained 
Usage 
Download SAM weights (vit_h ~2.5GB or vit_b ~375MB) from the official GitHub. Load with

  STEP 08  Final Output — Colored Change Map   [ Weeks 11–12 ] 
  The final step combines all masks from previous steps into a single visualized output. We overlay colored regions on the 
original RGB image: red for buildings detected on farmland, yellow for vegetation that changed but no building detected, 
and green for stable agricultural areas. The result is exported as GeoTIFF (georeferenced), PNG (visual), and a summary 
JSON report.

import numpy as np, cv2 
import rasterio, json 
from rasterio.features import shapes 
import geopandas as gpd 
 
def colorize_output(rgb, change_map, agri_mask, building_mask): 
    output = rgb.copy() 
    alpha = 0.55  # overlay transparency 
 
    # Buildings on farmland = RED 
    encroach = (building_mask>0) & (agri_mask>0) & (change_map>0) 
    output[encroach] = (output[encroach]*(1-alpha) + 
                        np.array([255,40,40])*alpha).astype(np.uint8) 
 
    # Changed vegetation (no building) = YELLOW 
    veg_change = (change_map>0) & (agri_mask>0) & ~encroach 
    output[veg_change] = (output[veg_change]*(1-alpha) + 
                          np.array([255,220,0])*alpha).astype(np.uint8) 
 
    # Stable agriculture = GREEN tint 
    stable = (agri_mask>0) & (change_map==0) 
    output[stable] = (output[stable]*(1-alpha) + 
                      np.array([40,200,80])*alpha).astype(np.uint8) 
    return output 
 
colored = colorize_output(t2_rgb, change_map, agri_mask, building_mask)
