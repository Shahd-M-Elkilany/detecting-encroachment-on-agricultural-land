6 · Recommended Fixes

  1. Set dashboard default threshold to 0.29. The slider defaults to 0.73, showing Recall=0.273 and making the model look broken. The operating threshold is 0.29.
  2. Gate report generation on RF ≥ ALERT_THRESHOLD (0.40). site15 and site276 are below this threshold. They should render a “No Encroachment Detected” page, not a cluster report.
  3. Clearly label raw-change vs RF-classified outputs. The tile-level ΔNDVI mask (kemet1_ba_allchanges.png) shows all pixels above spectral thresholds — not the same as the RF verdict. Annotate both with their respective methodology.
  4. Standardise units to ha everywhere. Remove the km² display path from the HTML report template. Single unit = no confusion.
  5. Fix ΔNDVI colorbar to a shared scale (±0.5). The per-site sample strip autoscales per panel, making cross-site visual comparisons invalid.
  6. Auto-flag high-area sites for manual review. Sites where RF-classified change > 15% of tile area (~100 ha) likely contain seasonal vegetation noise. Add a “Requires Verification” badge for those sites in the index and reports.
