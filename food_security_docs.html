Buildings on Agricultural Land

An 8-step deep learning pipeline that processes multi-temporal Sentinel-2 satellite imagery to automatically detect illegal construction on farmland — at national scale, with zero manual review.

System Overview
What Does This System Do?

The system receives two satellite images of the same geographic area taken at different times — one before (T1) and one after (T2) — and produces a color-coded map identifying where agricultural land has been encroached upon by construction.

1–4

Preprocessing Phase

Download Sentinel-2 imagery via Google Earth Engine. Detect and remove cloud cover. Compute spectral vegetation and built-up indices.

Steps 01–04
5–7

AI Analysis Phase

Detect land-use changes using ChangeFormer. Segment agricultural areas with SegFormer-B4. Identify buildings with SAM (Segment Anything).

Steps 05–07
8

Output Phase

Combine all masks with Boolean logic into a final colored PNG, georeferenced GeoTIFF, GeoJSON polygon file, and a JSON statistics report.

Step 08
∞

Key Advantage

Uses only free and open satellite data. Pretrained models require no training from scratch. Covers any region with Sentinel-2 coverage globally.

Zero-cost data
Data Inputs
SAT Team → Pipeline: File Formats

The Satellite communication team delivers two GeoTIFF files per analysis request. These are the only external inputs the pipeline needs. Below is the exact specification.

ℹ️
Two files are required: T1 (before image, baseline period) and T2 (after image, monitoring period). Both must cover the same geographic area and use identical CRS / spatial resolution.

📄 GeoTIFF Specification

# ─── SENTINEL-2 GEOTIFF SPECIFICATION ────────────────────────────
# Required for both T1 (before) and T2 (after) image files

FORMAT: GeoTIFF (.tif / .tiff)
BANDS: 5 bands in this exact order:
  Band 1 → B2 (Blue, 490 nm)
  Band 2 → B3 (Green, 560 nm)
  Band 3 → B4 (Red, 665 nm)
  Band 4 → B8 (NIR, 842 nm)
  Band 5 → B11 (SWIR, 1610 nm)

PIXEL SIZE: 10 m/pixel (B11 must be resampled from 20m)
DATA TYPE: uint16 (0 – 10000, surface reflectance × 10000)
CRS: WGS 84 / UTM Zone [local zone, e.g. 36N for Egypt]
CLOUD COVER: < 20% over the Area of Interest
TIME GAP: T2 must be ≥ 6 months after T1 (to capture seasonal stability)

# ─── NAMING CONVENTION ──────────────────────────────────────────
T1 FILE: {AREA}_{YYYYMMDD}_T1_S2.tif
T2 FILE: {AREA}_{YYYYMMDD}_T2_S2.tif

# Example:
  NileDelta_20220301_T1_S2.tif
  NileDelta_20240901_T2_S2.tif

# ─── DELIVERY ────────────────────────────────────────────────────
METHOD: Google Drive shared folder OR local path
FOLDER: data/raw/T1/ and data/raw/T2/
MIN SIZE: AOI ≥ 5 km × 5 km (500 × 500 pixels at 10m)

Step 01 — Data Acquisition

  • Input: AOI polygon (lat/lon), date range (T1 and T2), cloud threshold < 20%
  • Process: Authenticate GEE → query COPERNICUS/S2_SR_HARMONIZED → median composite → export
  • Output: Two GeoTIFF files — T1 and T2, 5 bands, 10m/pixel
  • Code: src/step_01_data_acquisition/acquire.py
  • Runtime: 5–15 min (GEE export time)

Step 02 — Cloud Detection (U-Net + ResNet34)

  • Input: GeoTIFF tiles (1, 3, 512, 512) — RGB bands normalized with ImageNet mean/std
  • Process: Tile image → U-Net inference → sigmoid → threshold at 0.45 → stitch tiles
  • Output: Binary cloud mask (H×W) uint8 — 255=cloud, 0=clear
  • Code: src/step_02_cloud_detection/detect_clouds.py
  • Runtime: ~2–4 min per image (GPU)

Step 03 — Cloud Removal (Telea Inpainting)

  • Input: GeoTIFF (all bands) + binary cloud mask from Step 02
  • Process: For each band: normalize → cv2.inpaint(INPAINT_TELEA, radius=7) → stack bands
  • Output: Cloud-free GeoTIFF — all original bands reconstructed, same CRS/geotransform
  • Code: src/step_03_cloud_removal/remove_clouds.py
  • Limitation: Works best for cloud cover < 40%. Larger coverage may need DSen2-CR.

Step 04 — Spectral Indices

  • Input: Cloud-free T1 and T2 GeoTIFFs with NIR (B8), Red (B4), SWIR (B11), Green (B3)
  • Process: Pure NumPy arithmetic — NDVI, NDBI, MNDWI computed per pixel
  • Output: Three float32 arrays per image: NDVI [-1,1], NDBI [-1,1], MNDWI [-1,1]
  • Code: src/step_04_spectral_indices/compute_indices.py
  • Runtime: < 30 seconds (CPU)

Step 05 — Change Detection (ChangeFormer)

  • Input: T1 tensor (1,3,256,256) + T2 tensor (1,3,256,256) — ImageNet normalized
  • Process: Siamese Transformer encodes T1 and T2 separately → difference decoder → argmax
  • Output: Binary change map (H×W) — 255=changed pixels, 0=unchanged
  • Code: src/step_05_change_detection/detect_changes.py
  • Pretrained weights: weights/ChangeFormer_LEVIR.pth (manual download required)

Step 06 — Agriculture Segmentation (SegFormer-B4)

  • Input: T1 cloud-free RGB image (H,W,3) uint8
  • Process: SegformerImageProcessor → inference → logits (1,150,H/4,W/4) → upsample → argmax
  • Output: Binary agriculture mask (H×W) uint8 — 255=farmland (ADE20K class IDs: 9,29,92,94,96)
  • Code: src/step_06_agriculture_segmentation/segment_agriculture.py
  • ADE20K classes used: field=9, earth=29, grass=92, dirt=94, plant=96

Step 07 — Building Detection (SAM)

  • Input: T2 RGB image cropped to (change_map ∩ agri_mask) overlap region
  • Process: SamAutomaticMaskGenerator → filter by area (>100px), IoU threshold, stability score
  • Output: Building mask (H×W) uint8 + instance polygons + confidence scores
  • Code: src/step_07_building_detection/detect_buildings.py
  • Weights: weights/sam_vit_b_01ec64.pth (375 MB, manual download)

Step 08 — Final Output

  • Input: change_map + agri_mask + building_mask + T2 RGB from all previous steps
  • RED pixels: building_mask > 0 AND agri_mask > 0 AND change_map > 0
  • YELLOW pixels: change_map > 0 AND agri_mask > 0 AND NOT red_region
  • GREEN pixels: agri_mask > 0 AND change_map == 0
  • Outputs: final_colored.png, final_colored.tif, encroachment.geojson, report.json
Mathematics
Mathematical Equations

Every formula used in the pipeline — from spectral index computation to neural network loss functions.

📊 Step 04 — Spectral Indices

NDVI — Normalized Difference Vegetation Index
NDVI = NIR − RedNIR + Red = B8 − B4B8 + B4

Range: [−1, +1]. Values > 0.3 indicate healthy vegetation. High NDVI in T1 and low in T2 signals vegetation loss — potential construction site.

NDBI — Normalized Difference Built-up Index
NDBI = SWIR − NIRSWIR + NIR = B11 − B8B11 + B8

Range: [−1, +1]. Values > 0.1 indicate concrete, rooftops, or paved surfaces. High NDBI increase from T1→T2 confirms new construction.

MNDWI — Modified Normalized Difference Water Index
MNDWI = Green − SWIRGreen + SWIR = B3 − B11B3 + B11

Range: [−1, +1]. Used to mask out water bodies (canals, rivers) that could be misclassified as changed vegetation. Values > 0 = water.

🧠 Step 02 — Cloud Detection Loss Functions

Binary Cross-Entropy Loss
BCE = − [ y · log(ŷ) + (1−y) · log(1−ŷ) ]

y = ground-truth pixel label (0 or 1), ŷ = predicted probability after sigmoid. Penalizes confident wrong predictions heavily.

Dice Loss (Overlap-based)
Dice = 1 − 2 · |X ∩ Y||X| + |Y|

X = predicted cloud mask, Y = ground-truth mask. Handles class imbalance (clouds cover < 20% of pixels). Dice = 0 is perfect overlap.

Combined Training Loss
Ltotal = α · BCE + (1−α) · Dice

α = 0.5 (equal weighting). The combination gives both pixel-level accuracy and shape-level accuracy for cloud boundaries.

Sigmoid Activation (final layer)
σ(x) = 11 + e−x

Maps raw U-Net logits to [0,1] probability. Threshold at 0.45: if σ(x) ≥ 0.45 → cloud pixel. Threshold tuned for high recall.

📐 Evaluation Metrics

Intersection over Union (IoU / Jaccard Index)
IoU = |A ∩ B||A ∪ B| = TPTP + FP + FN

Primary metric for all segmentation and detection tasks. IoU = 1 means perfect overlap. Used for cloud, agriculture, and building masks.

F1 Score (Change Detection)
F1 = 2 · Precision · RecallPrecision + Recall

Primary metric for ChangeFormer evaluation. Balances false positives (wrong change detected) and false negatives (missed change).

Final Map — Boolean Logic
🔴 RED = Bmask ∧ Amask ∧ Cmap
🟡 YELLOW = Cmap ∧ Amask ∧ ¬RED
🟢 GREEN = Amask ∧ ¬Cmap

B=building, A=agriculture, C=change. Boolean AND (∧), NOT (¬). RED takes priority: a pixel cannot be both RED and YELLOW.

🌾 SegFormer-B4 (Step 06)

Hierarchical Vision Transformer encoder with a lightweight All-MLP decoder. Outputs 150 semantic class logits, from which agriculture-related classes are extracted.

Agriculture Classes Extracted:
field [9] earth [29] grass [92] dirt [94] plant [96]

Pretrained: ADE20K (150 classes) · Source: nvidia/segformer-b4-finetuned-ade-512-512 · Auto-downloads via HuggingFace

🏠 SAM — Segment Anything (Step 07)

Meta AI's Vision Transformer-based universal segmentation model. In automatic mask generation mode, it generates thousands of segment proposals and filters them by area, stability, and IoU confidence.

Model: ViT-B (375 MB) or ViT-H (2.5 GB)
Pretrained on: SA-1B — 1.1 billion masks from 11M images
Mode: Zero-shot (no satellite fine-tuning needed)
Filter: Area > 100px, stability_score ≥ 0.92

Paper: arXiv 2304.02643 · No fine-tuning required · Works on any domain

Output Formats
Pipeline Output Files

Step 08 produces four output files, each serving a different downstream use case.

📸 final_colored.png

Visual PNG showing the full extent of the AOI with RED/YELLOW/GREEN color overlay blended at 60% opacity on top of the T2 satellite image. Ready for presentations and reports.

PNG · RGB · Human review

🌍 final_colored.tif

Same as the PNG but in GeoTIFF format with full CRS and geotransform metadata preserved. Can be loaded into QGIS, ArcGIS, or Google Earth Pro with exact geographic positioning.

GeoTIFF · Georeferenced · QGIS-ready

📍 encroachment.geojson

Vector polygon file of every detected encroachment area (RED regions). Each polygon includes properties: area_m2, area_ha, confidence_score, and centroid coordinates. Can be sent to authorities.

GeoJSON · Vector · Actionable

📊 report.json

Machine-readable statistics: total agricultural hectares monitored, total encroachment hectares, % farmland lost, number of buildings detected, and per-class pixel counts.

JSON · Statistics · API-friendly
